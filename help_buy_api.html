如何购买/开通 API（OpenAI & DeepSeek）

完全小白也能看懂：准备什么 → 去哪里 → 按什么顺序点 → 拿到 API Key → 放到 SummarizerX / 麦乐可 AI 摘要阅读器 里使用。

  • 🧾 按步骤做即可
  • 🔐 API Key 就像密码
  • 💳 支付方式：国际信用/借记卡等
  • 📘 我想买 OpenAI
  • 🟠 我想买 DeepSeek

一、OpenAI（ChatGPT API 等）

目标：开通计费 & 生成 API Key

  1. 访问 OpenAI 平台，注册/登录账号（需要邮箱，可能需要手机号验证）。
    注意：部分地区/号码可能无法验证，需更换网络或使用可接收短信的手机号。
  2. 开通计费（Billing）：进入 Billing， 添加信用卡/借记卡等支付方式，完成开通。
    • OpenAI 采用按量计费，先绑定支付方式，按实际使用计费。
    • 若显示 insufficient_quota / billing_not_active，通常是未完成计费开通或额度不足。
  3. 生成密钥：打开 API Keys 页面 → 点击 Create new secret key → 起个名字 → 复制保存。
    小贴士：只会显示一次，务必复制保存到安全位置（如密码管理器）。
  4. 在扩展设置中粘贴：打开 SummarizerX 设置页 → 选择 OpenAI → 粘贴刚复制的 Key → 保存。
可选：10 秒自测（命令行）

在终端运行下列命令（替换 YOUR_KEY）。若返回 200，说明 Key 可用：

curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer YOUR_KEY"

二、DeepSeek

目标：在控制台创建 API Key 并开通用量/余额

  1. 访问 DeepSeek 控制台（或从 开发者文档 进入），注册/登录账号。
  2. 开通计费或充值：在控制台的 Billing/充值 或 Usage 区域，绑定支付方式或充值余额。
    不同地区/账号可能显示为“按量付费”或“余额充值”，按页面提示完成即可。
  3. 创建密钥：进入 API Keys 或 密钥管理 → 点击 Create → 复制保存。
  4. 在扩展设置中粘贴：打开 SummarizerX 设置页 → 选择 DeepSeek → 粘贴 Key → 保存。
可选：10 秒自测（命令行）

DeepSeek 兼容 OpenAI 风格 API。将 YOUR_KEY 替换为你的密钥：

curl https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer YOUR_KEY"

↑ 返回顶部

三、把 Key 放进「SummarizerX / 麦乐可 AI 摘要阅读器」

  1. 打开扩展设置 → 「AI 服务提供商」选择 OpenAI 或 DeepSeek（两者都可以保存，随时切换）。
  2. 将对应的 Key 粘贴到输入框 → 保存。
  3. 首次请求会很快返回「摘要」，稍后自动补全「可读正文」。若响应慢，多半是 Key 配额或网络的问题。
提示：扩展支持同时保存两个 Key，后续切换无需重复输入。

四、常见问题（FAQ）

Key 是什么？丢了怎么办？

Key 就像账号的“通行证”。一旦泄露，别人可能用你的额度。若怀疑泄露，请立刻在控制台撤销/删除旧 Key 并重新创建新 Key。

提示 “insufficient_quota / billing_not_active” 怎么办？
  • 确认已经完成计费开通（OpenAI 需绑定支付方式；DeepSeek 可能需要充余额）。
  • 检查是否消费超限或余额不足，必要时提升额度/继续充值。
请求能通，但摘要迟迟不出？
  • 网络质量或接口波动可能导致响应变慢，稍等即可。扩展采用“两段式加载”，先出摘要再补全文。
  • 若长时间无响应：检查控制台用量页面是否有错误日志；或尝试切换到另一个提供商测试。
我可以把 Key 给同事用吗？

不建议。更安全的做法是：为团队使用创建单独的 Key（不同备注名），并定期轮换；或使用各平台提供的组织/项目级别权限隔离。

如何更省钱？
  • 在扩展里优先使用“摘要后再清洗全文”的策略，减少不必要的大段调用。
  • 合理限制最长输出字数、模型选择（性价比更高的模型）等。
本页仅作指引，实际界面以各平台为准。请从官方控制台完成购买/开通与 Key 管理。
